Using a Timeout Notification node to automatically start a flow

IBM Knowledge Center
Learn how to use the Timeout Notification node to automatically start a flow after a regular configurable time interval.
  • Create
  • Prepare
  • Run

Import projects

When you click Import, two projects will be created in your workspace: An application named ExampleTimeoutNotification and a policy project named myPols.

ExampleTimeoutNotification provides a single message flow named TimeoutNotification. The flow is started using a Timeout Notification node which is configured to fire a flow propagation on an automatic regular basis every 20 seconds. The second node in the flow is a trace node which appends text to a trace file which records the time when the trace node was reached, the interval for flow propagation and the message count.

myPols is a policy project holding a single policy named T1. The policy won't be used to start with, but is provided ready for the extension to the basic scenario which is described on the Run tab.

Once imported using the link on the right, the application ExampleTimeoutNotification is ready to be deployed and tested. To deploy, you can use a provided BAR file, as described on the Prepare tab.

Prepare a trace directory, create a queue manager and integration node, and then deploy the message flow

The Trace node uses a trace directory and the Timeout Notification node requires MQ queues to operate, so execute the following steps to prepare your environment accordingly:

  1. The message flow's Trace node will attempt to write a trace file to the directory C:\temp. Create this directory if it does not already exist, or edit the Trace node in the message flow to use a different directory suitable for your runtime environment.
  2. Create the default App Connect Enterprise queues in IBM MQ by completing the following steps:
    1. Open the IBM App Connect Enterprise Console. If you don't already have one, create and start a queue manager:
      crtmqm qmgr
      strmqm qmgr
    2. Next create the queues by typing the following command:
      server\sample\wmq\iib_createqueues qmgr
      where
      qmgr is the queue manager name.
  3. Create an integration node which is associated with the above queue manager by completing the following steps:
    1. In the console window, create an integration node by typing the following command:
      mqsicreatebroker TESTNODE -q qmgr
    2. Start the integration node by typing the following command:
      mqsistart TESTNODE
    3. Create an integration server associated with the integration node by typing the following command:
      mqsicreateexecutiongroup TESTNODE -e default
  4. Back in the Toolkit, drag and drop deploy the provided BAR file named TimeoutNotification.bar which you will find inside the ExampleTimeoutNotification application.

Use the Flow Exerciser to test the tutorial scenario

  1. Open TimeoutNotification.msgflow.
    • Click the Flow Exerciser icon to start testing the flow (when challenged you do not need to redeploy the flow again).
    • This flow does not need you to send it a message to drive it. Simply wait for about 20 seconds for a flow propagation to occur. If you are impatient, you can check the trace directory which was configured on the Prepare tab for the presence of the trace file named TraceTimeoutNotification.txt. Once the flow has fired, the trace node should contain an entry that looks like this:
      The flow propagation is from the Timeout Notification node with count number 1
      The flow propagation has reached the Trace node at 2019-08-04 20:18:56.974640
      The flow was started by Timeout Notification node with the identifier 'ID1'
      The flow Timeout Notification node currently has its interval set to 20 seconds
    • After you close the dialog, the path taken through the messageflow is highlighted. Using a text editor, open the trace file C:\temp\TraceTimeoutNotification.txt.
    • Next, reconfigure the TimeoutNotification node. Change the Unique Identifier property to the value {myPols}:T1. This means that the TimeoutNotification node will take its interval from the policy named T1 which resides in the policy project named myPols. Drag and drop deploy the policy project named myPols, and then drag and drop deploy the application ExampleTimeoutNotification (or do both these actions together by reconfiguring the TimeoutNotification.bar which you used earlier). The policy has been set up with the shorter interval of 5 seconds. You can experiment with different intervals by simply redeploying the policy project after updating the value of its property named Timeout (seconds).

This tutorial has shown the Timeout Notification node automatically start a flow on a regular timed interval.
